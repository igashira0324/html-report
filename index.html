LFM2-2.6B-Exp: エッジAIのための純粋強化学習

LFM2-2.6B-Exp 研究インフォグラフィック

研究概要

Liquid AIは、実験的な言語モデルであるLFM2-2.6B-Expを開発しました。これは26億パラメータを持ち、従来の教師ありファインチューニング手法とは異なり、純粋な強化学習を用いてトレーニングされている点が特徴です。

主な機能とアーキテクチャ:

  • ハイブリッドアーキテクチャ: 短距離ゲート付き畳み込みとグループ化クエリ注意 (GQA) を統合。
  • コンテキスト長: 32,768トークンという実質的な長さをサポート。
  • 最適化: 理解度、指示の遵守、知識の想起の向上を目指します。

パフォーマンスベンチマーク:

  • IFBench: DeepSeek R1-0528（LFM2-2.6B-Expより263倍大きいモデル）を上回る性能。
  • GPQA（大学院レベルの知識質問応答）: 約42%を達成。
  • IFEval（厳格な指示遵守）: 88%以上を達成。
  • GSM8K（数学的推論）: 82%以上を達成し、Llama3.23BやGemma3シリーズなどのモデルを凌駕。
  • モデルの小さなサイズと高いパフォーマンスが対比されています。

推奨されるユースケース:

  • エージェントワークフロー
  • データ抽出
  • RAG（検索拡張生成）
  • 創造的なライティング
  • 多ターン対話

デプロイメントと利用可能性:

  • 効率的なローカルデプロイメント: スマートフォン、ラップトップ、IoTデバイスなどのエッジデバイス向けに設計されています。
  • オープンソース: 重みはHugging Faceプラットフォームで利用可能です。
  • 多言語サポート: 英語、アラビア語、中国語、フランス語、ドイツ語、日本語、韓国語、スペイン語をサポート。
  • 制限事項: 非常に知識集約的なタスクやプログラミングスキルを必要とするタスクには推奨されません。

技術詳細

  • モデルサイズ: 2,569,272,320 パラメータ。
  • レイヤー数: 30（畳み込み22 + 注意8）。
  • 精度: bfloat16。
  • 語彙サイズ: 65,536。
  • トレーニング予算: 10兆トークン。
  • ライセンス: LFM Open License v1.0。
  • アーキテクチャ: 乗算ゲートと短畳み込みを特徴とするハイブリッドモデルで、10個の二重ゲート短距離LIV畳み込みブロックと6個のグループ化クエリ注意（GQA）ブロックを備える。
  • 事前学習混合: 約75%英語、20%多言語、5%コードデータ。
  • トレーニングアプローチ: 大規模SFT、長さ正規化を伴うカスタムDPO、反復モデルマージ、検証可能な報酬を伴う強化学習。
  • チャットテンプレート: ChatMLライクな形式。
  • ツール使用: 特殊トークンを使用して、関数定義、関数呼び出し、関数実行、最終回答生成をサポート。

実行とファインチューニング

  • Transformers: transformers v4.55+が必要。Pythonコードの例が提供されています。
  • vLLM: vLLM v0.10.2+が必要。Pythonコードの例が提供されています。

  • ファインチューニング: SFT (Unsloth, TRL) およびDPO (TRL) 用のノートブックが利用可能。

パフォーマンス比較（ブログ投稿データ）

  • トレーニング効率: 前世代LFMより3倍高速。
  • 推論速度: Qwen3と比較してCPUでのデコードとプリフィルが最大2倍高速。
  • ベンチマーク性能: 知識、数学、指示追従、多言語機能において、同規模のモデルを凌駕。
    • LFM2-1.2BはQwen3-1.7B（47%大きい）と競合。
    • LFM2-700MはGemma 3 1B ITを上回る。
    • LFM2-350MはQwen3-0.6BおよびLlama 3.2 1B Instructと競合。
  • LLM-as-a-Judge: LFM2-1.2BはLlama 3.2 1B InstructおよびGemma 3 1B ITよりも大幅に好まれ、Qwen3-1.7Bと同等。LFM2-700MはQwen3-0.6Bよりも好まれる。
  • 推論スループット（CPU）: ExecuTorchおよびLlama.cpp上でのモデルサイズに対するプリフィルおよびデコード速度のパレートフロンティアを支配。LFM2-700MはQwen-0.6Bよりも一貫して高速。

LFM2アーキテクチャの詳細

  • Liquid Time-constant Networks (LTCs) および Linear Input-Varying (LIV) オペレータに基づいている。
  • STARと呼ばれるニューラルアーキテクチャ探索エンジンを使用。
  • 言語モデリング評価（50以上の内部評価）およびアーキテクチャ効率（ピークメモリ、組み込みSoC CPUでのプリフィル+デコード速度）の直接測定のために修正されたSTARアルゴリズム。
  • 最終アーキテクチャ: 16ブロック（10個の二重ゲート短距離LIV畳み込み、6個のGQAブロック）、それぞれにSwiGLUとRMSNormを搭載。
  • ターゲットハードウェア（組み込みSoC CPU）と最適化されたカーネルライブラリのため、短畳み込みに焦点を当てる。

LFM2のトレーニング

  • 3つのモデルサイズがトレーニングされた: 350M、700M、1.2Bパラメータ。
  • 約75%英語、20%多言語、5%コードからなるコーパスから10兆トークンでトレーニング。
  • 多言語は日本語、アラビア語、韓国語、スペイン語、フランス語、ドイツ語に焦点を当てる。
  • LFM1-7B教師モデルからの知識蒸留。
  • 事前学習中にコンテキスト長を32kに拡張。
  • 後処理: 多様な下流タスクでの大規模SFT、セミオンラインデータを用いたカスタムDPO、反復モデルマージ。
